# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please:

### ✅ DO:

- Email us privately at **security@bookiebeats.com**
- Provide detailed steps to reproduce the issue
- Allow reasonable time for us to address the issue before public disclosure

### ❌ DON'T:

- Post security issues publicly in GitHub Issues
- Attempt to access other users' data or accounts
- Perform testing on production systems without permission

## Security Measures

### Code Security

- All code is open source and auditable
- TypeScript for type safety
- No external dependencies in content scripts
- Minimal permission requests

### Data Security

- No sensitive data collection
- All settings stored locally only
- No network requests to external servers
- Content Security Policy restrictions

### Browser Security

- Follows browser extension security best practices
- Manifest V3 compliance for enhanced security
- Isolated execution contexts

## Response Timeline

- **Initial Response**: Within 48 hours
- **Status Updates**: Weekly until resolved
- **Resolution**: Target 30 days for critical issues

Thank you for helping keep Bookie Beats Companion secure!
